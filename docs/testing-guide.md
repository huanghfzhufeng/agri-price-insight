# 测试文档

## 1. 文档目标

本文档用于规范本项目的测试范围、测试方法、验收标准和测试记录方式，确保系统最终能够作为毕业设计完整交付，而不是“能跑但不可验证”的演示程序。

## 2. 测试目标

本项目测试需要覆盖四类目标：

- 功能正确性
- 数据正确性
- 页面可用性
- 系统稳定性

## 3. 测试范围

### 3.1 后端测试范围

- 健康检查接口
- 系统基础字典接口
- 仪表盘数据接口
- 价格查询接口
- 统计分析接口
- 预测预警接口
- 登录鉴权接口
- 数据采集任务接口
- 导出接口

### 3.2 前端测试范围

- 导航切换
- 页面加载
- 查询筛选
- 图表渲染
- 异常提示
- 空状态展示
- 登录流程
- 管理页操作

### 3.3 数据测试范围

- 采集结果完整性
- 清洗结果正确性
- 去重逻辑正确性
- 异常值标记正确性
- 聚合统计结果正确性
- 预测结果格式与评价指标输出正确性

## 4. 测试策略

### 4.1 单元测试

目标：

- 验证核心函数逻辑正确

重点测试对象：

- 数据清洗函数
- 指标计算函数
- 环比同比函数
- 异常值识别函数
- 预测结果后处理函数

建议工具：

- pytest

### 4.2 接口测试

目标：

- 验证 API 输入输出正确

重点检查：

- 参数校验
- 响应格式
- 错误码
- 空结果处理
- 分页结果

建议工具：

- pytest
- httpx / FastAPI TestClient

### 4.3 前端联调测试

目标：

- 验证页面和接口联动是否正常

重点检查：

- 查询条件是否正确传参
- 表格数据是否与接口一致
- 图表数据是否与后端返回一致
- 错误态是否正常显示

### 4.4 手工验收测试

目标：

- 面向答辩演示场景验证用户体验

重点检查：

- 页面切换是否流畅
- 表单是否易用
- 图表是否清晰
- 不同屏幕宽度下布局是否可用

## 5. 环境要求

### 5.1 开发测试环境

- Python 3.11
- Node.js 20+
- npm 10+
- SQLite / MySQL

### 5.2 浏览器

- Chrome 最新版
- Edge 最新版

## 6. 模块测试用例设计

## 6.1 系统模块

### 用例 1：健康检查接口

- 前置条件：后端服务已启动
- 操作步骤：访问 `GET /api/v1/system/health`
- 预期结果：返回 `200` 且 `status=ok`

### 用例 2：系统选项接口

- 前置条件：数据库已写入演示数据
- 操作步骤：访问 `GET /api/v1/system/options`
- 预期结果：返回产品、市场、分类列表

## 6.2 数据查询模块

### 用例 1：按农产品名称查询

- 操作步骤：输入“大蒜”后发起查询
- 预期结果：返回仅包含“大蒜”的记录

### 用例 2：按日期范围查询

- 操作步骤：设置开始日期和结束日期后查询
- 预期结果：结果集日期均位于选定范围内

### 用例 3：无数据情况

- 操作步骤：输入不存在的品名
- 预期结果：页面显示“暂无符合条件的数据”

## 6.3 仪表盘模块

### 用例 1：概览指标加载

- 操作步骤：进入系统概览页
- 预期结果：显示统计卡片、趋势图、涨跌榜

### 用例 2：接口异常场景

- 操作步骤：关闭后端后刷新页面
- 预期结果：页面显示加载失败提示

## 6.4 预测预警模块

### 用例 1：预测接口返回结构

- 操作步骤：访问 `GET /api/v1/alerts/forecast`
- 预期结果：返回 `history`、`forecast`、`mape`、`rmse`

### 用例 2：切换 7 天 / 30 天 / 90 天预测

- 操作步骤：前端切换时间范围
- 预期结果：页面图表和说明文案同步变化

### 用例 3：预警列表显示

- 操作步骤：进入预测预警页
- 预期结果：显示预警级别、品种、市场、阈值和提示信息

## 6.5 登录模块

### 用例 1：正确登录

- 前置条件：管理员账号存在
- 操作步骤：输入正确用户名和密码
- 预期结果：登录成功并跳转首页

### 用例 2：错误密码

- 操作步骤：输入错误密码
- 预期结果：提示登录失败

## 6.6 数据采集模块

### 用例 1：采集任务成功

- 操作步骤：执行一次手动采集
- 预期结果：生成原始记录和标准记录，任务日志为成功

### 用例 2：目标站点异常

- 操作步骤：模拟官方页面访问失败
- 预期结果：任务日志记录失败原因

## 7. 数据正确性校验

必须额外验证以下内容：

- 原始表记录数 >= 标准表记录数
- 标准表不存在同品种、同市场、同日期重复数据
- 单位标准化后数值范围合理
- 日均价处于最小价和最大价之间
- 同比和环比结果可通过抽样人工复核

## 8. 性能与稳定性要求

毕业设计不需要做到大规模压测，但至少应满足：

- 单页接口平均响应时间 < 2 秒
- 查询接口在演示数据规模下稳定返回
- 前端页面首屏可正常渲染
- 连续多次查询不出现崩溃
