# 答辩演示脚本

## 开场

各位老师好，我的毕业设计题目是《基于 Python 的农产品价格数据分析与可视化系统的设计与实现》。本系统围绕官方价格数据采集、清洗入库、统计分析、趋势预测与可视化展示展开。

## 第一部分：项目目标

本项目解决的问题是：农产品价格数据分散、人工分析成本高、缺乏直观展示和短期预测能力。因此我设计了一套完整的 Web 系统，实现从数据采集到图形展示的闭环。

## 第二部分：技术架构

后端采用 FastAPI，数据库支持 SQLite 和 MySQL，ORM 使用 SQLAlchemy，前端采用 React、Vite、Tailwind 和 ECharts。系统还实现了登录鉴权、任务日志、月报归档和阈值管理。

## 第三部分：功能演示

### 1. 登录系统

先演示管理员登录，验证系统具备基础权限控制。

### 2. 系统概览

在首页可以看到今日更新数据、异常预警数量、重点品类趋势和涨跌排行。

### 3. 数据查询

在数据查询页可以按品种、市场、时间范围查询价格数据，并支持分页和 CSV 导出。

### 4. 统计分析

统计分析页提供同比、环比、区域对比、波动率排行和异常值识别。

### 5. 预测预警

预测预警页展示未来价格走势、置信区间和模型评价指标，同时列出当前异常预警。

### 6. 系统管理

系统管理页展示真实采集任务日志、原始文章、数据源配置、阈值设置和月报归档。

## 第四部分：项目特点

- 使用官方数据源
- 不是静态报表，而是完整系统
- 同时兼顾业务数据和图片展示
- 具备分析与预测能力
- 具备较好的毕业设计展示效果

## 结束语

以上就是我的系统演示。后续我还可以继续扩展更复杂的预测模型、更多数据源和更完整的角色权限体系。
